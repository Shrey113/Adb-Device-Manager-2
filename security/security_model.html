1. Overview

ADB Device Manager is built with a security-first philosophy. Everything operates locally — no cloud servers, no external transmission, and full offline functionality.

No Cloud Servers

All communication stays local

No External Transmission

Data never leaves your network

Offline-First Design

3. Communication Channels

ADB Mode (USB)

Direct USB connection via Android Debug Bridge. Uses local scrcpy server for mirroring. Cannot access Internet.

App Mode (WiFi)

Local network via WiFi/hotspot. Uses WebSocket/HTTP over LAN with mDNS discovery. TLS encryption where supported.

INTERNET / NETWORK Local WiFi communication
FOREGROUND_SERVICE Background connection

6. Local Storage Security

Minimal configuration stored locally: window preferences, device profiles, and license status. No sensitive data (SMS, notifications, calls, contacts) is stored persistently.

7. Premium License Security

Local-only license validation. No cloud connection, no personal data collected. License key stored locally on your Windows PC.

8. Threat Model

✓ Protects Against

  • Unauthorized remote access
  • Data mining and profiling

⚠ Limitations

  • Insecure public WiFi
