# Privacy Policy

> ADB Device Manager — Privacy Policy & Data Practices

---

## 1. Introduction

ADB Device Manager is designed to operate **fully offline**. All communication happens directly between your Windows PC and Android device using USB or local Wi-Fi.

- **No personal data is uploaded** to external servers
- **No data is transmitted** to third parties
- **No data is stored** on remote servers
- **No data is shared** with anyone

The app processes data only on your devices.

---

## 2. What Data the App Accesses and Why

To provide its features, the app may access the following data types:

### Contacts

- **Purpose:** Display caller names and message sender identities
- **Storage:** Not stored

### SMS Messages

- **Purpose:** Display and allow sending SMS through Windows app
- **Storage:** Not stored

### Call Logs / Phone State

- **Purpose:** Detect incoming calls and display call history
- **Storage:** Not stored

### Notifications

- **Purpose:** Mirror Android notifications to Windows
- **Storage:** Stored temporarily in memory only

### Files / Storage

- **Purpose:** Local file transfers between devices
- **Storage:** Files saved only where you choose
- **Upload:** Never uploaded

### Device Model & Device Info

- **Purpose:** Identify connected devices
- **Storage:** Stored only as local app preferences
- **Personal Data:** Contains no personal data

### Audio / Microphone

- **Purpose:** Audio during screen mirroring
- **Processing:** Real-time only

---

## 3. Data Storage

The app stores very minimal information.

### Permanent Local Storage

Stored only on the user's device:

- App settings
- Device connection preferences
- Local premium license validation

**No personal data** such as SMS, contacts, call logs, or notifications is saved.

### Temporary Storage

Used only during active sessions:

- Notification previews
- Screen mirroring video frames
- Audio stream buffers

Temporary data is deleted immediately when the session ends.

### External Storage / Cloud Storage

The app does **NOT** use:

- Cloud uploads
- Remote servers
- Online backups
- External analytics

All data remains on your device.

---

---

## 5. Permissions Explanation

The Android companion app may request the following permissions:

| Permission | Purpose |
|------------|---------|
| **Notification Access** | Mirror notifications to Windows |
| **READ_SMS / SEND_SMS** | Display and send SMS messages |
| **READ_CONTACTS** | Show caller names |
| **READ_CALL_LOG / READ_PHONE_STATE** | Detect calls and show call history |
| **RECORD_AUDIO** | Audio during screen mirroring |
| **READ/WRITE STORAGE** | File transfer between devices |
| **INTERNET / NETWORK / WIFI STATE** | Local Wi-Fi communication only |
| **FOREGROUND SERVICE** | Keep connection stable in background |
| **RECEIVE_BOOT_COMPLETED** | Optional background restoration |

**All permissions are used only for offline functionality.**
No permission is used to collect or transmit personal data.

The app does not knowingly process or store data from children.
